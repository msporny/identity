This document is setup to document use cases and requirements for identity solutions on web on mobile and document proposed standards associated with identity on the web. The use cases and requirements were gathered examining those that already exist on native and proprietary solutions today as well as gathering any statistics that exist on usage patterns. This document was setup in response to the email chain started 29 January 2014 and continuing in the W3C Web and Mobile IG mailing list.

Proprietary solutions that integrate web applications into the underlying operating system have been used in the wild for a few years now, so consolidation towards a standardized solution could potentially benefit users, developers, and browser vendors.

Edits on this document is happening now. If you have any thoughts or comments about the content, please either join the public-webmob mailing list or file a bug on GitHub. All contributions welcome.

Introduction

Many native applications on mobile devices are giving users a personalised experience or allowing them to access services based on users proving their identity to some level of assurance. An application may simply need one method of knowing the user interacting with the application is different from another user interacting on another device. On the other hand it may require a detailed level of identification assurance to allow a user to access sensetive or personal data. Depending on the situation an application may understand the user as identified by the device they are using, by their operator, or identify them based on a login system (or another method).

Some native and proprietary applications are also using mobile devices as a second form of identification, in '2 (or more) factor authentication'.

Web applications often provide users with login systems to access records and data. Many web players, operating system vendors and browser vendors have created their own proprietary login and identity systems. There has been some move forward with standardisation, however, the IETF created the OAuth protocol, which has now developed into the Open ID Connect protocol as part of the Open ID Foundation. In this document we will try to discover the use cases and requirements for identity on the web on mobile based on native application login and identity real-world scenarios. We will detail use cases around assurance levels, user experience flow and security, and investigate some of the tecnical aspects which make identity on the web on mobile slightly more difficult than with native applications.

Levels of Assurance

Some applications carry very little sensetive data and will require a user to login purely in an effort to retrieve this (e.g. past scores or levels played for a game). Other applications require a much higher level of assurance, and possibly some proof of the users identity, to gain access to the app as the app holds a lot on sensetive and personal information (such as banking details of health records).
